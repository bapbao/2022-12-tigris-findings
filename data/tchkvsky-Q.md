Use of deprecated `ERC20Permit` file

### Handle

tchkvsky

### Scope

[StableToken.sol](https://github.com/code-423n4/2022-12-tigris/blob/588c84b7bb354d20cbca6034544c4faa46e6a80e/contracts/StableToken.sol#L4)

### Vulnerability details

The Openzeppelin extension file imported is deprecated. Since [`EIP-2612`](https://eips.ethereum.org/EIPS/eip-2612) is final, OZ's [`draft-ERC20Permit.sol`](https://github.com/OpenZeppelin/openzeppelin-contracts/blob/master/contracts/token/ERC20/extensions/draft-ERC20Permit.sol) has been [deprecated](https://github.com/OpenZeppelin/openzeppelin-contracts/blob/3d7a93876a2e5e1d7fe29b5a0e96e222afdc4cfa/contracts/token/ERC20/extensions/draft-ERC20Permit.sol#L6) and might be removed from the library in the near future. This could lead to unwanted behaviour in future implementations of the `StableToken` contract.

### Recommendation

Use the more stable [`ERC20Permit.sol`](https://github.com/OpenZeppelin/openzeppelin-contracts/blob/master/contracts/token/ERC20/extensions/ERC20Permit.sol) instead.
